## Durable lessons

- Preserve fallback paths when browser or API automation fails.
- Draft first when reputation, money, security, or ambiguity is involved.
- Measure meaningful responses rather than raw activity.
- Promote operating rules only after outcomes support them.
- Treat credentials exposed in chat or committed to Git as compromised.
