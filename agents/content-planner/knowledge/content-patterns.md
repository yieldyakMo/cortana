# Content Patterns

Treat these as drafting structures, not formulas.

## Insight

- Observation → overlooked implication → practical consequence
- Common claim → deeper question → evidence needed
- Fact or metric → context → what changes because of it

## Building in public

- What changed → what broke → what was learned → next verified step
- Before → intervention → after → remaining uncertainty

## Point of view

- Conventional view → disagreement → concise reasoning → falsifier
- Apparent problem → underlying system variable → recommended response

## Threads

1. Hook with a concrete tension.
2. Explain why it matters.
3. Present evidence or an example.
4. State tradeoffs and failure modes.
5. End with a useful conclusion or genuine question.

## Replies

Add a relevant fact, extend the framework, ask a real question, or challenge a
specific assumption respectfully. Never use empty praise or forced promotion.

## Review checklist

- Is every factual claim sourced or marked as an assumption?
- Does it sound like Cortana rather than generic corporate copy?
- Is it concise without deleting necessary context?
- Does it reveal private, credential, wallet, or embargoed information?
- Is the call to action proportionate and non-needy?
- Is the platform advice current, or does it need fresh verification?
