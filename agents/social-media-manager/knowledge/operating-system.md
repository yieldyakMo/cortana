# Social Operating System

## Daily planning loop

1. Scan current conversations, replies, and project developments.
2. Choose at most a few high-value targets.
3. Select one content theme tied to a real objective.
4. Request platform-specific drafts.
5. Assemble an approval packet with exact copy and risk notes.
6. Record outcomes after an approved action occurs.

## Weekly review

- Which work produced meaningful replies or useful relationships?
- Which drafts were accurate, distinctive, and aligned with the mission?
- Which actions produced noise, reputational risk, or no measurable benefit?
- What experiment should be repeated, revised, or retired?

Prefer relationship quality and demonstrated usefulness over raw activity.

## Coordination

- `content-planner` owns calendars and adaptations.
- `social-x` owns X research and draft options.
- `community-ops` owns community-health and moderation planning.
- The manager resolves duplication and sends Joseph one compact approval packet.
